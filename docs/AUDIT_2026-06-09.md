# Skein Codebase Audit — 2026-06-09

Full audit of the Skein umbrella (compiler, runtime, CLI, LSP, tests, docs).
Findings were produced by parallel deep reviews of each app plus direct
verification of every critical claim against the source. Items marked
**[verified]** were confirmed by reading the cited code; the rest come from
the per-app reviews.

## Health snapshot

| Check | Result |
|-------|--------|
| `mix compile --warnings-as-errors` | ✅ Clean, all 4 apps |
| `mix test` | ✅ 1,343 tests + 182 properties, **0 failures** (compiler 820+74, runtime 442+108, cli 48, lsp 33) |
| `mix format --check-formatted` | ❌ **25 files unformatted** (incl. `parser.ex`, `analyzer.ex`, `core_erlang.ex`, `tool.ex`) |
| TODO/FIXME/HACK comments | ✅ None found |
| Committed build artifacts / risky deps | ✅ None; umbrella config clean |
| Property + stateful (PropCheck) coverage | ✅ Strong — lexer/parser/analyzer/codegen StreamData suites, 5 statem suites |

Overall: the project is in good shape — well-tested, well-structured, clean
compile. The significant risks are concentrated in the runtime's concurrency
and input-handling patterns, plus one violation of the project's own
structured-error contract.

## Critical

### C1. Atom exhaustion from external input (runtime) [verified]
Atoms are never garbage-collected; creating them from untrusted input can
crash the VM.

- `apps/skein_runtime/lib/skein/runtime/router.ex:154` —
  `defp method_atom(other), do: String.downcase(other) |> String.to_atom()`.
  The HTTP method comes **straight off the wire**, so any client can mint
  unlimited atoms. This is the only remotely-triggerable instance and the
  most urgent fix (return 405 for unknown methods instead).
- `apps/skein_runtime/lib/skein/runtime/store.ex:139` — `String.to_atom("skein_store_#{table_name}")`
- `apps/skein_runtime/lib/skein/runtime/memory.ex:209` — same pattern for namespaces
- `apps/skein_runtime/lib/skein/runtime/migration_gen.ex:24,80,88` and
  `ecto_schema.ex:55,63,69` — table/field names to atoms (compile-time-ish
  input, lower risk, still worth a whitelist).

### C2. TOCTOU race in ETS table creation (runtime) [verified]
The check-then-create pattern crashes with `ArgumentError` when two processes
race, and several sites have **no** rescue:

```elixir
# store.ex:142-148 — no rescue
defp ensure_table(ets_table) do
  if :ets.whereis(ets_table) == :undefined do
    :ets.new(ets_table, [:named_table, ...])
  end
  :ok
end
```

Affected: `store.ex:142`, `memory.ex:212`, `store_ecto.ex:172`,
`event_store.ex:45`, `timer.ex:228`, `idempotent.ex:119`, `tool.ex:29`.
MEMORY.md already documents this exact pitfall ("Always wrap `:ets.new` in
try/rescue") but it is applied inconsistently. Better: create all tables in a
supervised init path.

### C3. Structured-error contract violated: missing `fix_code` (compiler) [verified]
CLAUDE.md design constraint #5: every compiler error must carry `fix_hint`
**and** `fix_code`. 12 error sites omit `fix_code`:

- `parser.ex` lines 289, 303, 424, 591, 603, 615, 1434, 2358, 2377, 2396, 2424
  (e.g. the central `expect/3` at 2358 — so *every* "expected token" error
  ships without `fix_code`)
- `lexer.ex:239-246` (E0001 unexpected character)

This is flagged critical because agent-consumable errors are a stated core
feature, and `expect/3` is the most common error path in the parser.

## Major

### M1. Lazily self-started GenServers race (runtime)
`ensure_started/0` in `process.ex:124`, `queue.ex:152`, `topic.ex:192`,
`schedule.ex:172`, `timer.ex:238` does `unless Process.whereis(__MODULE__),
do: start_link()`. Two concurrent effect calls can both attempt the start;
one gets `{:error, {:already_started, _}}` or a crash. Start these under the
application supervisor instead (also fixes ownership/restart semantics).

### M2. `mix format --check-formatted` fails on 25 files [verified]
Violates the project's own first style rule, and implies formatting isn't
enforced in CI. One `mix format` commit + a CI check fixes this permanently.

### M3. CLI crashes on malformed flags [verified]
`apps/skein_cli/lib/skein_cli.ex`:
- `parse_flags` line 418: `String.to_integer(port_str)` — `--port abc` raises
- `parse_trace_flags` line 465: same for `--last foo`
- trace `--kind` line 452: `String.to_existing_atom(kind_filter)` raises on
  any kind that doesn't already exist as an atom

Use `Integer.parse/1` and a kind whitelist; return structured CLI errors.

### M4. SQL built by string interpolation (runtime)
`event_store/sqlite_backend.ex` interpolates `#{@table_name}` into CREATE
TABLE / INSERT strings. Safe today (module attribute constant), but the
pattern is one refactor away from injection. Same family: `store_ecto.ex:140`
builds `field(r, ^key)` from unvalidated filter keys — validate against the
schema's declared fields.

### M5. Unbounded EventStore growth (runtime)
`event_store.ex` appends every event to the `:skein_events` ETS table with no
eviction. Fine for dev; a guaranteed memory leak for long-running services.
Needs TTL/size-based eviction or offload to the SQLite backend.

### M6. LSP app skips the documentation conventions
`apps/skein_lsp/lib/skein/lsp/server.ex` (~23 public/callback functions) has
zero `@spec`/`@doc`; `semantic_tokens.ex` and `symbols.ex` are mostly
undocumented. Test coverage is also thin (33 tests, no full request/response
integration tests). The LSP app is the clear convention outlier.

### M7. `examples/hello_llm.skein` has no test
13 of 14 examples are covered in `examples_test.exs`; `hello_llm.skein` is
the gap.

## Minor

1. **Codegen `gen_var` uses the process dictionary** (`core_erlang.ex:2124`)
   for its variable counter — unsafe if compilation ever becomes concurrent
   within a process, and leaks state between compilations. Thread a counter
   through scope instead.
2. **Lexer `column_after_comment/2` always returns 1** (`lexer.ex:440-444`),
   so location reporting after a `--` comment with trailing content can be
   wrong; interpolation close at `lexer.ex:315` has a similar off-by-one risk.
3. **`Analyzer.resolve_type/2` (public, `analyzer.ex:501+`) lacks `@spec`.**
4. **No property tests for `schema_gen.ex`** — CLAUDE.md mandates StreamData
   coverage for codegen; `core_erlang` has it, schema generation doesn't.
5. **Deprecated `EventLog` still carries a full test suite**
   (`event_log_test.exs`, `event_log_property_test.exs`) — migrate to
   EventStore or document the back-compat intent.
6. **Queue/Topic/Schedule share ~90% of dispatch + ensure_started logic** —
   extract a shared helper.
7. **Router swallows body-read failures** (`router.ex:140-146` assigns `""`
   on `{:more, ...}` and `{:error, ...}`) and the 500 handler logs nothing —
   add logging; chunked bodies silently become empty.
8. **LLM client**: streaming buffers all chunks in memory; `:httpc` is used
   without explicit timeouts/pooling; verify Req exception messages can't
   leak the Anthropic API key (`anthropic_backend.ex:171`).
9. **Idempotent TTL**: `ttl_ms()` is re-read at sweep vs. check time —
   cache it or document the constant-TTL assumption.
10. **Docs staleness**: `CLAUDE.md` project tree still lists the deleted
    `docs/IMPLEMENTATION_PLAN.md`; one Elixir 1.19 typing warning fires in
    `hover_provider_test.exs:68` (always-true `!= nil` comparison).

---

## Suggested fix order

1. `router.ex:154` method whitelist (remote DoS vector) + the other
   `String.to_atom` sites (C1)
2. Centralize ETS/GenServer startup under the runtime supervisor (C2, M1)
3. Add `fix_code` to the 12 parser/lexer errors (C3)
4. `mix format` everything + add a CI format check (M2)
5. Harden CLI flag parsing (M3)
6. EventStore eviction policy (M5)
7. The minor list opportunistically, starting with LSP docs/specs (M6) and
   the `hello_llm` example test (M7)

## Resolution log — 2026-06-10

All findings were worked through on branch `claude/audit-2026-06-09-issues-p93ywh`.

### Critical
- **C1 — Resolved.** `Router` whitelists HTTP methods and returns 405 for
  unknown ones. `Store` and `Memory` were refactored to single static ETS
  tables (`:skein_store` keyed by `{table, id}`, `:skein_memory` keyed by
  `{namespace, key}`), eliminating dynamic atom creation entirely.
  `EctoSchema`/`MigrationGen` validate table/field names against an
  identifier whitelist (`EctoSchema.validate_name!/2`) before any
  `String.to_atom`.
- **C2 — Resolved.** The one remaining unguarded site (`tool.ex`) got the
  try/rescue treatment; `store.ex`/`memory.ex` now create one static table
  each with rescue. (The audit list was partially stale: `store_ecto.ex`,
  `event_store.ex`, `timer.ex`, and `idempotent.ex` already had rescues.)
- **C3 — Resolved.** All 12 parser/lexer sites now emit `fix_code`
  (`expect/3` via `token_text/1`, `unexpected_token_error/3` via
  `default_fix_code/1`); regression tests added. Follow-up completed: the
  analyzer's ~30 remaining error sites also carry `fix_code` now, enforced
  by a contract sweep (`analyzer_fix_code_test.exs`) that exercises every
  analyzer error path and asserts both `fix_hint` and `fix_code` are set.

### Major
- **M1 — Resolved.** Process/Queue/Topic/Schedule/Timer are children of
  `SkeinRuntime.Application`; `ensure_started/0` is now a race-tolerant
  fallback for `--no-start` environments.
- **M2 — Resolved.** `mix format` applied; `.github/workflows/ci.yml` added
  (format check + `compile --warnings-as-errors` + full test suite). There
  was previously no test CI at all.
- **M3 — Resolved.** `--port`/`--last` use `Integer.parse` with validation
  and return structured errors; trace `--kind` compares kinds as strings
  (no atom conversion); `main.ex` handles trace errors.
- **M4 — Resolved.** `StoreEcto.query` validates filter keys against the
  schema's declared fields. `SqliteBackend` has a compile-time guard
  asserting `@table_name` stays a plain SQL identifier.
- **M5 — Resolved.** EventStore evicts oldest events past
  `:event_store_max_events` (default 100k); documented in moduledoc and
  site docs.
- **M6 — Resolved.** Added an 11-test GenLSP.Test integration suite (full
  initialize/didOpen/diagnostics/symbols/hover/completion/semantic-tokens
  round-trips). Writing it exposed and fixed two real bugs: `lsp.assigns`
  was read as a map (it is the Assigns agent pid in gen_lsp 0.11) and
  `Server` had no `start_link/1` for its generated `child_spec`. Public
  API modules already carried `@doc`/`@spec`; `start_link/1` is documented.
- **M7 — Resolved.** `hello_llm.skein` covered (compile, phases,
  transitions, phase handler behavior).

### Minor
1. **Partially resolved.** `gen_var` counter is reset at each `generate/1`
   entry, fixing cross-compilation leakage and determinism. It stays in
   the process dictionary deliberately (threading it through ~21 sites is
   churn without benefit; one compilation per process at a time).
2. **Resolved.** `skip_to_eol` counts skipped characters; `:eof` column is
   correct after trailing comments. Test added.
3. **Resolved.** `@spec` added to `Analyzer.resolve_type/2`.
4. **Resolved.** `schema_gen_property_test.exs` added (7 StreamData
   properties).
5. **Resolved.** Both EventLog suites carry moduledocs declaring the
   back-compat intent.
6. **Deferred deliberately.** Queue/Topic/Schedule dispatch differs
   semantically (fan-out vs single-consumer, capability checks on Topic);
   extraction would obscure more than it saves.
7. **Resolved.** Router logs body-read failures and handler crashes (with
   stacktrace) before responding 500.
8. **Resolved (partially).** API key is redacted from transport exception
   messages; `:httpc` got a connect timeout (it already had a request
   timeout — audit claim was stale). Streaming still buffers chunks by
   design (codegen emits a no-op stream callback).
9. **Resolved.** Constant-TTL assumption documented on `ttl_ms/0`.
10. **Resolved.** CLAUDE.md tree updated to the actual layout (no
    IMPLEMENTATION_PLAN.md, full runtime/example listings); hover test
    typing warning fixed; site runtime docs updated for the single-table
    memory design and EventStore eviction.
